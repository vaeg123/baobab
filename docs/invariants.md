# BAOBAB — Invariants système

## Les 8 invariants non négociables

### 1. Immuabilité des Legal Atoms
Toute connaissance juridique est immuable et versionnée.
Un Legal Atom ne se modifie jamais — toute évolution crée une nouvelle version.
L'identifiant `CI.CIMA.CODE.ART260.V2022` est permanent et unique.

### 2. Traçabilité totale des raisonnements
Toute conclusion produite par BAOBAB est explicable.
Chaque output inclut : sources primaires + exceptions + niveau de confiance.
Aucun "boîte noire" — tout raisonnement est auditable.

### 3. Isolation des données clients
Les données d'un client n'alimentent jamais un modèle concurrent.
Chaque tenant est isolé logiquement et physiquement.
Violation = rupture contractuelle immédiate.

### 4. Rattachement aux sources primaires
Toute obligation identifiée est rattachée à une source juridique primaire.
Pas d'obligation "inventée" — tout est ancré dans le corpus légal.
Source minimale : corpus + article + version + territoire.

### 5. Séparation raisonnement / décision
BAOBAB produit du raisonnement. La décision appartient à l'humain.
Aucune action irréversible n'est déclenchée automatiquement.
L'output est toujours une recommandation, jamais une exécution.

### 6. Versionnement du droit
Toute évolution législative génère de nouveaux atoms, jamais de modifications.
L'historique juridique est préservé indéfiniment.
Un dossier de 2019 peut être retraité avec le droit de 2019.

### 7. Souveraineté des données africaines
Les données traitées par BAOBAB restent en Afrique (ou sur des serveurs conformes RGPD/CEDEAO).
Priorité aux hébergeurs africains certifiés.

### 8. Neutralité régulatoire
BAOBAB ne prend pas parti dans les conflits d'interprétation.
En cas d'ambiguïté, toutes les interprétations possibles sont présentées.
Le juriste client tranche — BAOBAB informe.
